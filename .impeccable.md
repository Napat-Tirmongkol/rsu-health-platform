# Design Context — RSU Healthcare Services Portal

## Design Context

### Users
Superadmin: technical/administrative staff at Rangsit University. Uses the portal regularly to manage vaccination campaigns, user records, system settings, and deployment (git pull). Context is desktop, focused work sessions — not casual browsing. They need clarity, speed, and confidence that actions are correct.

### Brand Personality
**3 words**: Clinical · Refined · Trustworthy

The portal should feel like a premium internal tool — the kind you'd expect from a well-funded healthtech SaaS. Not flashy. Not consumer-facing. Calm authority.

### Aesthetic Direction
- **Theme**: Light mode. Clean white backgrounds with the clinic green as a grounding accent — not splashed everywhere, but present where it matters.
- **Green palette**: #2e9e63 / #3bba7a — keep as primary brand color. Use it purposefully: active states, key actions, positive status. Not as decoration.
- **Typography**: Prompt (already in use). Lean on weight contrast (800 for labels/headings, 600 for body, 400 for secondary). No gradient text.
- **Layout**: Asymmetric, left-aligned. Sidebar + content shell. Generous breathing room in content areas. Not centered-everything.
- **Density**: Medium — enough whitespace to feel premium, but information-dense enough for power users.
- **Anti-references**: No glassmorphism, no hero metric cards with gradient accents, no identical card grids, no cyan/purple AI palette.

### Design Principles
1. **Calm authority** — Every screen should feel like it's in control. No visual noise, no competing CTAs.
2. **Green with purpose** — Use #2e9e63 only for interactive states, positive status, and primary actions. Not as background fill.
3. **Weight = hierarchy** — Let font weight (not size alone) do the heavy lifting for hierarchy. Headings: 800, labels: 700, body: 500.
4. **Flat depth** — Use 1px borders and subtle background tints instead of shadows and blur effects.
5. **Thai-first copy** — All labels, status text, and microcopy in Thai. English only for technical terms (Git Pull, Status, ID).
